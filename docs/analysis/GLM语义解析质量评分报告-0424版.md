## 典型误报模式

GLM 的核心问题是**不理解"股票代码"的约束**，经常输出新闻关键词而非真实上市 ticker：

| 类型 | AI 推荐 | 问题 |
|------|---------|------|
| energy | `['OIL', 'Energy']` | 'OIL' 不是上市代码（应为 USO/XLE） |
| geo_political | `['IRAN', 'USGOV']` | IRAN 是国家名, USGOV 不是 ticker |
| geo_political | `['IRAN']` | 同上 |
| geo_political | `['UPS', 'pharma sector ETF']` | UPS 有效但混杂非标准描述 |
| tech | `['JASO', 'ATES', 'BEKE', 'ZB', 'YH']` | 部分可能已退市或虚构 |
| tech | `['GOOGL', 'GOOG']` | 有效 ticker 但验证器不认 |
| tech | `['PYPL', 'V', 'SQ']` | 有效 ticker 但验证器不认 |
| tariff | `['TWE', 'BEAM']` | 可能为特定标的但验证器不认 |
| tariff | `['SCOT', 'KYST']` | 同上 |
| other | `['AAPL', 'GOOGL', 'MSFT']` | 真实有效 ticker 但被标记误报 |
| energy | `['OIL', 'XLE']` | 'OIL' 无效，XLE 是 ETF |

> **注意**: 部分被标记为"误报"的 ticker（如 AAPL, GOOGL, PLTR, QCOM, PYPL, V, SQ）实际上是纳斯达克/NYSE 上市的有效代码。这说明**验证白名单可能不全**，存在误杀。

---

## 根本问题分析

1. **Prompt 约束不足**: GLM 输出 `OIL`、`IRAN`、`USGOV` 等非 ticker 字符串，说明 system prompt 中"只输出纽交所/纳斯达克上市股票代码"的约束执行不到位
2. **验证白名单不完整**: 有效 ticker（AAPL, GOOGL, PLTR 等）被标记误报，说明 `ticker_truth_source` 数据可能不完整
3. **板块识别优于个股识别**: 板块质量均分 44.0 显著高于个股质量均分 10.5，AI 对"这是什么板块"的判断比对"这是什么股票"的判断更准确
4. **AI 信心分与真实质量严重脱节**: AI 平均信心 79.7，但个股质量均分仅 10.5，AI 完全无法自知其错误

---

## 后续建议

1. **优化 system prompt**: 增加更严格的股票代码约束，禁止输出非 ticker 词汇
2. **补充验证白名单**: 确保 AAPL, GOOGL, PLTR, QCOM 等常见 ticker 在验证列表中
3. **分类型调优**: tariff 类表现好无需大幅改动；tech 和 geo_political 类需重点优化
4. **运行当前版本对比**: 当前项目 `事件驱动交易模块`（git `6911bf0`, Stage6）代码已有优化，可启动 live 模式对比 GLM 表现
