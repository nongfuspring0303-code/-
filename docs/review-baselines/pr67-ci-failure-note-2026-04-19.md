# PR #67 CI 失败说明（2026-04-19）

## 结论
- 本次 CI 未通过不是单测/语法错误，而是健康门禁失败：
  - `CANARY_SOURCE_HEALTH` 判定为 `RED`
  - 失败项：`p95_latency_ms_1h=3285.94 > 3000.0`

## 指标解释
- `p95` 表示：最近 1 小时请求延迟按从小到大排序后，95% 请求的耗时上界。
- `p95 = 3285.94ms` 的含义是：
  - 95% 的请求耗时 `<= 3.28594s`
  - 仍有 5% 请求更慢。

## 为什么触发失败
- 当前门禁阈值为 `3000ms`。
- 实际 `3285.94ms` 超过阈值，因此 `PROJECT_CHECK` 失败并退出 `exit code 1`。

## 关于调阈值
- 提高阈值（例如调整到 `3300ms` 或以上）可以让这一条检查通过。
- 但这属于“放宽门禁”，不代表真实延迟性能已经改善。
