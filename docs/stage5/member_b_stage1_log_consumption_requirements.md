# Member B Stage1 Log Consumption Requirements
**Version**: v1.0  
**Date**: 2026-04-22  
**Role**: Member B review / sign-off for Stage1 evidence logs  
**Scope**: Stage1 minimal evidence logs only. This file defines the smallest consumable log contract for mapping acceptance and scorecard reuse.

---

## 1) B-side mandatory log files

B must be able to consume these 5 log files directly:

1. `raw_news_ingest.jsonl`
2. `market_data_provenance.jsonl`
3. `decision_gate.jsonl`
4. `replay_write.jsonl`
5. `execution_emit.jsonl`

---

## 2) Required fields by file

### 2.1 `raw_news_ingest.jsonl`
Required:
- `trace_id`
- `event_trace_id`
- `request_id`
- `batch_id`
- `event_id`
- `event_hash`
- `headline`
- `detected_at`
- `source_rank`

B may read these fields directly.

### 2.2 `market_data_provenance.jsonl`
Required:
- `trace_id`
- `event_trace_id`
- `request_id`
- `batch_id`
- `event_id`
- `event_hash`
- `market_data_source`
- `market_data_present`
- `market_data_stale`
- `market_data_default_used`
- `market_data_fallback_used`
- `validation_state`

B may read these fields directly.

### 2.3 `decision_gate.jsonl`
Required:
- `trace_id`
- `event_trace_id`
- `request_id`
- `batch_id`
- `event_id`
- `event_hash`
- `final_action`
- `final_reason`
- `gate_output`
- `output_gate`
- `semantic_event_type`
- `sector_candidates`
- `ticker_candidates`
- `a1_score`
- `theme_tags`
- `tradeable`
- `opportunity_count`

Directly readable by B:
- `trace_id`, `event_trace_id`, `request_id`, `batch_id`, `event_id`, `event_hash`
- `final_action`, `final_reason`, `semantic_event_type`, `a1_score`, `tradeable`, `opportunity_count`

Stable nested reads allowed:
- `gate_output.final_action`, `gate_output.reason`
- `output_gate.blockers`, `output_gate.reason`
- `sector_candidates` and `ticker_candidates` when already emitted as top-level summary arrays
- `theme_tags` when already emitted as top-level summary array

### 2.4 `replay_write.jsonl`
Required:
- `trace_id`
- `event_trace_id`
- `request_id`
- `batch_id`
- `event_id`
- `event_hash`
- `final_action`
- `action_card`

B may use `event_hash` and trace keys to join with ingest / provenance / gate evidence.

### 2.5 `execution_emit.jsonl`
Required:
- `trace_id`
- `event_trace_id`
- `request_id`
- `batch_id`
- `event_id`
- `event_hash`
- `order`
- `execution_receipt`

B may use `event_hash` and trace keys to join execution back to the evidence chain.

---

## 3) B-side minimum field set for acceptance

For mapping acceptance, B must be able to read:

- `trace_id`
- `event_hash`
- `semantic_event_type`
- `sector_candidates`
- `ticker_candidates`
- `a1_score`
- `theme_tags`
- `tradeable`
- `opportunity_count`

If a field is not present as a top-level field in `decision_gate.jsonl`, B may only consume it when it is available as a stable nested field with the same semantics. Temporary inference from unrelated logs is not allowed.

---

## 4) Direct-read vs stable-derivation rules

### Must be directly readable
- trace keys: `trace_id`, `event_trace_id`, `request_id`, `batch_id`, `event_id`, `event_hash`
- gate outcome: `final_action`, `final_reason`
- B summary fields: `semantic_event_type`, `sector_candidates`, `ticker_candidates`, `a1_score`, `theme_tags`, `tradeable`, `opportunity_count`

### May be read from stable nested objects
- `gate_output` and `output_gate` details in `decision_gate.jsonl`
- provenance booleans in `market_data_provenance.jsonl`
- replay / execution join details inside `action_card`, `order`, or `execution_receipt` when trace keys are already present

### Not acceptable
- deriving sector/ticker/A1/theme_tags from ad hoc runtime heuristics
- reading them from transient upstream objects that are not persisted into the stage1 logs

---

## 5) B sign-off conditions

B may sign off Stage1 only when all of the following are true:

- the 5 evidence files are generated by the runtime path
- every file carries consistent trace keys
- `decision_gate.jsonl` includes the B-side summary fields
- `trace_id` and `event_hash` can join raw ingest, provenance, decision gate, replay, and execution
- B can consume sector / ticker / A1 / theme_tags without reconstructing them from unstable paths
- the Stage1 logs are sufficient for mapping acceptance and scorecard reuse
