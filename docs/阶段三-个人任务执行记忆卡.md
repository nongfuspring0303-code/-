## 2. 开工前硬规则（必须先做）
1. 先读契约再改代码：`schemas/*.json` + `module-registry.yaml`。
2. 首次接手预检（协议第15章）：
   - `python3 -m pytest -q`
   - `PYTHONPYCACHEPREFIX=/tmp/pycache python3 scripts/system_healthcheck.py`
3. 若预检失败：先修复环境/基线，再开发。

## 3. 分支与提交规范
1. 从 `integration` 拉新后新建任务分支：`feature/<owner-task>`。
2. 提交必须带任务号（如 `T4.2`）。
3. 小步提交，单次只做一类变更。

## 4. 契约变更联动规则
- 默认四联动：
  - `schemas/*.json`
  - `tests/*`
  - `module-registry.yaml`
  - 文档（README/任务文档）
- 若变更 `event_update/sector_update/opportunity_update`：执行五联动（额外补前端消费契约说明）。
