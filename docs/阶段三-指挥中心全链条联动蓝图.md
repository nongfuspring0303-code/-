# EDT 阶段三：指挥中心全链条联动蓝图（修订版）

## 一、目标定义（你真正要看到的结果）

本阶段的目标不是“跑通模块”，而是做出一个可以实时观察的投研指挥中心：

1. 新闻变化后，板块热度能立即变化（利好/利空方向明确）。
2. 板块变化后，优质股票池自动重排并给出理由。
3. 股票池中自动筛出建仓机会（多头/空头都支持）。
4. 全链路结果可视化、可追溯、可回放。

核心原则不变：**AI 可建议，规则可执行，风控可否决。**

---

## 二、产品形态（一个界面看全链路）

建议落地为一个三栏联动实时界面（Web）：

### 左栏：新闻事件雷达
- 实时新闻流（时间、来源、标题、严重度、证据分）。
- 每条新闻显示：`trace_id`、`event_id`、`narrative_state`。
- 点击某条新闻后，中栏和右栏立即切换为该事件上下文。

### 中栏：板块传导热力图
- 以板块卡片/热力图展示“受影响方向 + 强度 + 置信度”。
- 颜色语义固定：利好绿色、利空红色、中性灰色。
- 同步显示传导链：宏观因子 -> 板块 -> 主题。

### 右栏：优质股机会池
- 只显示“优质股票候选池”内的标的，并按机会分排序。
- 每只股票展示：
  - 方向建议（LONG/SHORT/WATCH）
  - 入场区间（支撑/阻力附近）
  - 风险提示（相关性、流动性、疲劳度）
  - 最终动作（EXECUTE/WATCH/BLOCK/PENDING_CONFIRM）

## 三、产品化补齐（前端层 / 实时中间层 / 实盘执行层）

为避免系统停留在“脚本可跑但体验原始”的状态，阶段三明确补齐以下三层：

### 3.1 前端层（可视化指挥中心）
- 目标：把复杂 JSON 决策转为可读、可比、可追溯的操作界面。
- 交付：`canvas/index.html` 三栏联动页面（新闻雷达 / 板块热力图 / 优质股机会池）。
- 关键能力：
  - 同一 `trace_id` 一键联动展示
  - 状态变化可视化（WATCH -> EXECUTE / BLOCK）
  - 盘中时间线与盘后回放视图

### 3.2 实时中间层（事件总线与推送）
- 目标：把“单次命令执行”升级为“持续监听 + 实时推送”。
- 交付：实时事件总线（WebSocket/事件流）与三类标准消息：
  - `event_update`
  - `sector_update`
  - `opportunity_update`
- 关键能力：
  - 消息按 `trace_id` 有序到达
  - 消息丢失可重放（按时间窗/trace 检索）
  - 前端断线后可自动恢复订阅

### 3.3 实盘执行层（Broker 适配与对账闭环）
- 目标：从 `paper` 模式平滑过渡到真实券商执行。
- 交付：在 `BrokerAdapter` 抽象下实现 `LiveBroker`（券商待定后接入）。
- 关键能力：
  - 订单生命周期状态机（accepted/partial/filled/rejected/cancelled）
  - 重启恢复与对账（reconcile）
  - 幂等与风控硬闸不可绕过（含人工确认）

## 四、联动机制（News -> Sector -> Stock -> Decision）

### 步骤1：新闻到事件（A层）
- 模块：`NewsIngestion` + `EventObjectifier` + `EventEvidenceScorer`
- 输出：标准化事件对象（含证据分、时效分、一致性分、严重度）
- 关键要求：
  - 去重聚类（同一事件多源合并）
  - 异常来源降权
  - 失败时有 fallback 新闻源

### 步骤2：事件到板块（B层前半）
- 模块：`ConductionMapper` + `NarrativeStateRecognizer`
- 输出：板块影响向量（方向、幅度、叙事状态）
- 关键要求：
  - 明确利好/利空方向，不允许“只有分数没有方向”
  - 支持叙事阶段（initial/continuation/decay/invalid）

### 步骤3：板块到优质股（B层后半）
- 模块：`MarketValidator` + `AISignalAdapter` + 优质股池筛选器
- 输出：候选股票列表（附原因）
- 关键要求：
  - 优质池来源可配置（白名单/基本面评分）
  - 候选池按“确定性+流动性+风险”三维排序

### 步骤4：优质股到建仓机会（C层）
- 模块：`SignalScorer` + `RiskGatekeeper` + `PositionSizer` + `ExitManager` + `ExecutionAdapter`
- 输出：最终动作与执行回执
- 关键要求：
  - 同时支持多头与空头机会
  - 高风险必须进入 `PENDING_CONFIRM`
  - AI超时/错误必须触发 G7 安全降级

## 五、界面联动协议（前后端契约）

为保证“连贯通畅”，前端订阅三类实时消息（WebSocket/事件流）：

1. `event_update`
   - 字段：`trace_id`, `headline`, `severity`, `evidence_score`, `narrative_state`
2. `sector_update`
   - 字段：`trace_id`, `sector`, `direction`, `impact_score`, `confidence`
3. `opportunity_update`
   - 字段：`trace_id`, `symbol`, `signal`, `entry_zone`, `risk_flags`, `final_action`

联动规则：
- 同一 `trace_id` 的消息必须按时间有序渲染。
- 左栏新闻选中后，中栏/右栏仅展示该 `trace_id` 视图。
- 若后续状态变更（如 WATCH -> EXECUTE），界面要有状态动画和变更日志。

## 六、判定“直观且靠谱”的验收标准

### 可视化体验
- 新闻点击后，板块变化渲染延迟 <= 1s。
- 板块切换后，股票池刷新延迟 <= 1s。
- 单事件链路可完整回放（新闻 -> 板块 -> 股票 -> 决策）。

### 决策质量与安全
- 利好与利空场景能输出不同动作（非同质化）。
- 高风险场景触发 BLOCK 或 PENDING_CONFIRM（不可绕过）。
- AI故障场景触发 G7 降级（WATCH/BLOCK），不能直接 EXECUTE。

### 审计可追溯
- 每条决策都有 `trace_id`、`decision_summary`、`reasoning`、`Decision Card`。
- 支持按 `trace_id` 检索历史版本与执行回执。

### 成员A（情报与事件层）

1. A-1 事件到板块映射底座 `[待开发]`
   - 交付：`configs/sector_impact_mapping.yaml`
   - 验收：能输出板块方向与影响强度（`sector_update`）

2. A-2 新闻聚类与去重增强 `[待开发]`
   - 交付：`NewsIngestion` 去重窗口与聚类策略升级
   - 验收：新闻洪峰下同义事件不刷屏

3. A-3 传导链层级数据 `[待开发]`
   - 交付：`configs/conduction_chain.yaml`
   - 验收：可展示“宏观 -> 板块 -> 主题”链路

### 成员B（策略与机会层）

1. B-1 优质股票池筛选器 `[待开发]`
   - 交付：`configs/premium_stock_pool.yaml` + 筛选逻辑
   - 验收：机会池仅输出优质池内股票

2. B-2 多空分化机会评分 `[待开发]`
   - 交付：`opportunity_score` 规则与排序逻辑
   - 验收：利好/利空输入输出可区分（LONG/SHORT/WATCH）

3. B-3 机会卡片字段补全 `[待开发]`
   - 交付：机会卡字段（入场区间、风险旗标、理由摘要）
   - 验收：右栏卡片可直接用于投研讨论

### 成员C（编排、风控与可视化层）

1. C-1 实时总线与三类契约 `[待开发]`
   - 交付：`event_update`、`sector_update`、`opportunity_update` schema
   - 验收：前端可稳定订阅并渲染三类消息

2. C-2 三栏联动界面原型 `[待开发]`
   - 交付：`canvas/index.html`
   - 验收：点击新闻后中右栏按 `trace_id` 联动

3. C-3 风控与人工确认不可绕过 `[待开发]`
   - 交付：高风险拦截与 `PENDING_CONFIRM` 校验
   - 验收：未确认请求不得进入执行

### 接口冻结规则（强制）

字段变更必须四联动：`schemas` + `tests` + `module-registry.yaml` + 文档。

### 阶段联合验收标准

1. 联动延迟：新闻到板块 <= 1s，板块到机会池 <= 1s。
2. 决策分化：利好/利空差异化率 >= 80%。
3. 风控安全：高风险拦截率 100%，AI 故障降级正确率 100%。
4. 可追溯：`trace_id` 贯穿率 100%，支持按 `trace_id` 回放。

## 八、总结

这版蓝图的核心不是再加模块，而是把已有 A/B/C 能力组织成一个“看得见、跟得上、能执行、可复盘”的联动系统：

**新闻变 -> 板块变 -> 股票变 -> 决策变**，全过程统一在同一界面、同一 `trace_id` 下闭环。
