## 接口契约（冻结版）

### 契约 1: ConductionMapper 输出 → AssetValidator / PathAdjudicator 输入

```python
class ConductionOutput:
    """成员A的产出，成员B/C的输入"""
    trace_id: str                          # 事件追踪ID
    schema_version: str = "v1.1"           # 必须
    news_timestamp: str                    # ISO8601
    event_type_lv1: str                    # 一级事件类目
    event_type_lv2: str                    # 二级事件标签（映射主键）
    classification_confidence: float       # 分类置信度 [0,100]
    market_impact_confidence: float        # 市场影响置信度 [0,100]
    
    # === 成员A负责产出 ===
    shock_profile: Dict[str, List[str]]    # primary/secondary/narrative
    raw_macro_factor_vector: Dict[str, float]  # 9因子原始值，范围[-100, +100]
    
    # === 成员B负责填充 ===
    transmission_paths: List[Dict]         # 至少3条path（由path_router产出）
    impact_chain: List[Dict]               # 完整影响链（后台保留）
    dominant_path: Dict                    # 主路径裁决结果
    competing_paths: List[Dict]            # 竞争路径
    asset_validation: Dict                 # 资产验证结果
    macro_factor_vector: Dict[str, float]  # 生效因子（raw * 市场验证修正）
    
    # === 成员C负责填充 ===
    sector_rankings: Dict                  # 板块排序
    stock_candidates: List[Dict]           # <=5只
    action: str                            # TRADE/WATCH/NO_ACTION
    state_machine_step: str                # 状态机裁决步骤
    gate_reason_code: str                  # 闸门原因码
    score_breakdown: Dict                  # 分数拆解
```

- 10.3 决策状态机

### 具体任务

#### C1. sector_ranker 实现
- 新增 `transmission_engine/core/sector_ranker.py`
- 实现板块评分公式（4因子加权减惩罚）
- 输出 `primary_sector(<=1)` + `secondary_sector(<=1)` + `avoid_sector(<=2)`
- 输出结构严格遵循 **契约7**

#### C2. stock_exposure_model 实现
- 新增 `transmission_engine/core/stock_exposure_model.py`
- 实现个股评分公式（v1.2 借鉴：event_exposure/event_relevance/relative_strength/liquidity/risk_filter）
- 个股分类: first_order_carriers / leveraged_proxies / contaminated_names
- 仅从 `configs/premium_stock_pool.yaml` 选取
- 接入 `ticker x event_type_lv2` 暴露矩阵（新增 `configs/event_exposure_matrix.yaml`）
- 输出 `stock_candidates(<=5)`
- 输出结构严格遵循 **契约8**

#### C3. state_machine 实现
- 新增 `transmission_engine/core/state_machine.py`
- 实现 6 步决策状态机:
  1. Data Integrity Gate → 关键字段缺失 → NO_ACTION
  2. Mixed Regime Gate → mixed_regime=true → WATCH
  3. Asset Validation Gate → 未达 trade_min → WATCH/NO_ACTION
  4. Risk Gate → 流动性红区/风控红线 → NO_ACTION
  5. Trade Admission Gate → 全部满足 → TRADE
  6. Fallback → WATCH
- 后置门不能覆盖前置门的拒绝结论
- 输出严格遵循 **契约9**

#### C4. trade_adapter 实现
- 新增 `transmission_engine/core/trade_adapter.py`
- 整合 sector_ranker + stock_exposure_model + state_machine
- 输出最终行动: TRADE/WATCH/NO_ACTION
- 记录 gate_reason_code + score_breakdown

#### C5. opportunity_score.py 升级
- 修改 `OpportunityScorer.build_opportunity_update()`:
  - A/B/C 档改为配置驱动（读取 gate_policy.signal_grade.a_min/b_min）
  - 股票输出上限改为 5（优先读取 `gate_policy.signal_grade.max_stock_candidates`，缺失时回退 5）
  - 输出必须带 `gate_reason_code` / `score_breakdown` / `state_machine_step`

#### C6. shadow_comparator 实现
- 新增 `scripts/shadow_comparator.py`
- A轨(现网) vs B轨(新核) shadow 输出对比
- 输出一致性统计: action_match_rate, path_match_rate, sector_match_rate, score_delta_p95
- 阈值读取 `gate_policy.shadow.*`

#### C7. 配置模板
- 修改 `configs/premium_stock_pool.yaml`（优质股票池，补齐美股全板块覆盖）
- 新增 `configs/event_exposure_matrix.yaml`（ticker x event_type_lv2 暴露矩阵）

#### C8. 单元测试
- `tests/test_state_machine.py`: 6步优先级、拒绝不可覆盖
- `tests/test_sector_ranker.py`: 板块评分公式、数量限制

## 同伴检查与PR门禁（必须执行）

### 1. 评审角色分配（固定）
- 成员A PR：主审=成员B，副审=成员C
- 成员B PR：主审=成员C，副审=成员A
- 成员C PR：主审=成员A，副审=成员B

职责定义：
- 主审负责“是否可合并”的最终技术结论
- 副审负责风险补漏与兼容性复核

### 2. PR必须包含的证据（缺一不可）
- 变更文件清单（说明是否在本成员边界内）
- 契约映射说明（本PR对应契约编号及满足情况）
- 测试命令与结果（至少包含本成员负责测试）
- 配置读取证据（阈值来自 `gate_policy.yaml`，无硬编码）
- 兼容性说明（是否新增字段、是否保留旧字段）

### 3. 同伴检查清单（统一）
- 接口契约一致性：字段名、枚举、范围、必填项是否符合契约1-10
- 状态机优先级：是否满足“后置门不可覆盖前置拒绝”
- 配置单一真源：阈值是否全部配置读取，禁止魔法数字
- 越界改动检查：是否修改其他成员边界文件
- 测试有效性：新增测试是否覆盖本次行为变更
- 审计可追踪性：输出是否包含 `trace_id`、`gate_reason_code`、`score_breakdown`

### 4. 问题分级（统一口径）
- `BLOCKER`：契约破坏、阈值硬编码、越界改动、测试未过
- `MAJOR`：逻辑冲突、状态机顺序错误、关键审计字段缺失
- `MINOR`：命名、结构、可读性优化建议

处理规则：
- 存在任一 `BLOCKER`：禁止合并
- 存在未解决 `MAJOR`：禁止合并
- `MINOR` 可带改进建议，不阻塞合并

### 5. 合并门禁（必须全部满足）
- 主审与副审均已明确审批
- 无 `BLOCKER`，无未解决 `MAJOR`
- CI 全绿
- 本任务书中的契约兼容要求通过

#### 7.2 Commit Message 规范

格式：

`<type>(memberX/<scope>): <summary>`

- `feat`: 新功能

- `refactor`: 重构（不改行为）
- `test`: 测试
- `docs`: 文档
- `chore`: 其他维护项

- `feat(memberA/conduction): add shock_profile output with config-driven mapping`
