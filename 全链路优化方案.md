# 事件驱动交易模块阶段二 - 全链路优化方案

## 1. 目标与验收指标

- 新闻实时性：`P95 新闻滞后 <= 300s`，`P99 <= 900s`
- 可用性：主链路 `1h success_rate >= 0.98`
- 推荐有效性：有板块时 `opportunity_update` 非空率 `>= 90%`
- 稳定性：核心回归全绿（`pytest + verify_* + healthcheck`）

---

## 2. P0（优先，约1周）：数据源真实性与可用性闭环

### 2.1 新闻源分层

- `P0 主源`：新浪实时流（秒级，财经直播）✓ **已完成 (2026-04-10)**
  - API: `http://zhibo.sina.com.cn/api/zhibo/feed`
  - 延迟: ~100ms
  - freshness_lag: < 60s
- `P1 权威源`：SEC/官方公告（事件确权）
- `P2 兜底源`：Google Reuters RSS（当前环境可用）

### 2.2 Canary 门禁收敛

- 在 `canary_source_health` 中拆分统计：
  - 主源成功率
  - 回退成功率
  - 最终成功率
- 禁止把"回退成功"直接等价为"主源健康"。
- 过渡性阈值 override 保留开关，但必须在报告中显式标注"回退模式"。
- **已完成**: Canary 已切换到 Sina 作为主源，验证 GREEN

### 2.3 验收标准

- 可连续 24h 生成 canary 证据日志
- `system_healthcheck --mode prod` 能区分主源失败与回退成功
- canary 报告包含 `attempted_sources`、`used_source_url`、`freshness_lag_sec`

---

## 3. P1（1-2周）：A层语义与B层推荐增强

### 3.1 A层智能化（规则 + LLM 双轨）

- 规则先筛选，低置信度再调用 LLM（控成本）
- 统一输出字段：
  - `category`
  - `severity`
  - `sector_candidates`
  - `confidence`
  - `reasoning`
- 增加误判治理：
  - 错分黑名单（短期修复）
  - 强映射白名单（高确定性场景）

### 3.2 B层推荐能力增强

- 扩充 `premium_stock_pool.yaml`（每板块至少 8-15 个核心标的）
- 新增“事件类型 -> 个股特征”过滤（并购/财报/监管/地缘）
- 评分新增因子：
  - 流动性
  - 波动风险
  - 新闻-标的相关度
- 保留 `WATCH` 降级，同时在 `reasoning` 输出降级原因

### 3.3 验收标准

- 误判样本集（至少 50 条）通过率显著提升
- 有板块情况下 `opportunity_update` 非空率 >= 90%
- 推荐输出字段完整率 100%

---

## 4. P2（约2周）：C层可观测性与前端可解释性

### 4.1 前端观测增强

- 新增来源状态条：
  - 主源/回退源
  - 当前滞后秒数
  - 门禁状态（GREEN/YELLOW/RED）
- 事件卡固定展示：
  - `新闻发布时间`
  - `推送时间`
  - `源类型`
  - `trace_id`

### 4.2 可视化报表

- 增加 `canary_live dashboard`（读取 `logs/canary_live/*.json`）
- 输出 5m/30m/1h 三窗口趋势图

### 4.3 验收标准

- 页面可直接辨识当前是否在回退模式
- 同伴无需读源码即可定位“实时性问题来自哪一层”

---

## 5. P2 并行：质量门禁与发布准入

### 5.1 新增门禁脚本

- `verify_news_freshness.py`
- `verify_opportunity_non_empty_rate.py`

### 5.2 合并前必须通过

- `python3 -m pytest -q`
- `python3 scripts/system_healthcheck.py --mode prod`

### 5.3 验收标准

- CI 中门禁全绿才允许合并
- 任何门禁失败有明确错误归因

## 8. 关键风险与对策

### 风险1：外部源反爬导致主源波动

- 对策：主源 + 权威源 + 回退源三层设计，强制区分统计口径。

### 风险2：LLM 接入引入成本与不稳定性

- 对策：仅低置信度触发，设置调用上限与超时降级。

### 风险3：推荐“看起来有结果”但质量不高

- 对策：引入非空率、相关度、误判样本集指标，不以“有输出”替代“有效输出”。
