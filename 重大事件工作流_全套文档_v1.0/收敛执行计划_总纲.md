# 收敛执行计划_总纲 v2

> 核心思想：所有修改以给出最终交易动作为唯一目的。拒绝对动作质量无贡献的复杂度。

---

## 一、系统现状与目标差距

- **当前状态**：输出发散，结论模糊（如"值得关注"），缺乏市场验证硬拦截。
- **目标状态**：输出一张符合《附录 B》的 ActionCard JSON，包含 `trade_decision` / `trade_grade` / `position_tier`。
- **改造底线**：
  1. **NO A1 → NO TRADE**（市场优先于逻辑）
  2. **主路径收敛由规则系统负责，LLM 不裁决**（总纲 4.2 原则）
  3. **事件状态必须联动交易动作**（附录 F）

---

## 二、终极修改清单（按执行顺序）

### 阶段 1：LLM 输入层契约化（成员 A）

**目标**：强制 AI 提供结构化的事件"证据与要素"，不做路径裁决。

1. **修改 `ai_event_intel.py` / `ai_semantic_analyzer.py`**
   - 实施《LLM 规范 v2.1》
   - 输出：`a0_event_strength`（事件强度）、`expectation_gap`（预期差）、`event_state`（叙事阶段）
   - **LLM 只输出候选传导要素和概念证据，不输出最终主路径决策**
   - 必须有 fallback 降级机制（AI 超时/失败时返回保底 EventObject）

2. **修改 `ai_signal_adapter.py`**
   - 废弃旧 4 字段解析，对齐 v2.1 schema
   - 将 `expectation_gap` 映射为 A-1 权重乘数
   - 将 `event_state` 传递给下游状态映射模块

---

### 阶段 2：路径裁决与市场验证硬闸门（成员 B）

**目标**：落实附录 C（路径裁决）+ 附录 D（市场确认），这是 NO A1 → NO TRADE 的物理实现。

3. **修改 `transmission_engine` 核心（路径裁决）**
   - 实施《附录 C》收敛规则：默认目标 1 条主路径 + 0~2 条次级路径
   - 路径分层：P1（直接受益）> P2 强二级（保留白名单）> P3/P4 直接放弃
   - **保留强 P2 白名单机制**（如 P2 流动性/弹性显著优于 P1，允许升为备选）
   - 主路径评分维度：因果清晰度 / 传导距离 / 可交易性 / 流动性 / 市场优先承接概率

4. **修改 `market_validator.py`（市场三层闸门）**
   - 第一层：`macro_confirmation`（supportive / neutral / hostile）
   - 第二层：`sector_confirmation`（strong / medium / weak / none）
   - 第三层：`leader_confirmation`（confirmed / partial / unconfirmed / failed）
   - 最终结论：`a1_market_validation`（pass / partial / fail）
   - 同时输出：`positive_signals` / `negative_signals` / `reason_text`（供审计用）

---

### 阶段 3：主链编排与 ActionCard 输出（成员 C）

**目标**：实现附录 B + 附录 F，聚合所有结果，输出冷酷指令。

5. **修改 `workflow_runner.py`**
   - 引入《附录 F》状态映射矩阵：`event_state` × `a1_market_validation` → `trading_state`
   - 实施 A1 硬约束（代码层面写死，不只是注释）：
     ```
     a1_market_validation == fail → trade_decision ∈ {observe_only, avoid}，position_tier = none
     ```
   - 输出完整 ActionCard（六层：事件层 / 主路径层 / 市场确认层 / 动作层 / 风险层 / 一句话结论）

---

### 阶段 4：治理层（异步旁路，不阻塞主战链）

**目标**：实现附录 E，但绝不压主战链速度。

6. **归因落盘改为异步执行**
   - 主链返回 ActionCard 后，**异步/队列方式**写入复盘快照
   - 落盘字段：`replay_id` / `event_id` / 各模块评分快照 / ActionCard 完整内容
   - 不允许同步 I/O 出现在 Runner 的关键返回路径上

---

## 三、执行承诺

每一行代码的修改必须回答：**"这会让系统的最终买卖指令更清晰吗？"**

如果不能，不进主链。
