## 一、文档清单

### 1. 主文件
1. `重大事件工作流总纲与执行总规范_v1.0.md`
   - 作用：项目总纲、执行总规范、专项规范入口文件
   - 适合对象：项目负责人、全体成员、新伙伴

### 2. 附录文件
2. `LLM 新闻分析层设计与字段映射规范.md`
   - 作用：LLM 新闻分析层职责边界、字段 Schema、Prompt / Budget / Replay 规范
   - 适合对象：负责 LLM 层、Schema、联调、治理的人

3. `附录B_交易动作输出卡片规范.md`
   - 作用：定义最终交易动作卡片字段与输出口径
   - 适合对象：负责下游动作输出、卡片生成、交易语言标准化的人

4. `附录C_路径裁决规则规范.md`
   - 作用：定义主路径/副路径分层、路径收敛与降级规则
   - 适合对象：负责中游传导、路径裁决、机会筛选的人

5. `附录D_市场确认规范.md`
   - 作用：定义宏观 / 板块 / 龙头三闸门的确认逻辑
   - 适合对象：负责市场承接验证、A1 确认、风控门禁的人

6. `附录E_收益归因与回放规范.md`
   - 作用：定义收益归因、统一复盘口径、回写规则逻辑
   - 适合对象：负责复盘、统计、优化回写的人

7. `附录F_事件状态到交易状态映射规范.md`
   - 作用：定义事件阶段如何映射成观察 / 试仓 / 可做 / 加仓 / 减仓 / 回避
   - 适合对象：负责状态机、动作等级、执行衔接的人

## 三、文档关系说明

### 1. 总纲文件负责什么
主文件负责：
- 项目目标
- 主链路
- 职责边界
- 执行原则
- 专项规范索引

### 2. 附录文件负责什么
附录文件负责：
- 把总纲中的某个关键模块展开成专项执行规范
- 定义字段、规则、状态、映射、复盘口径
- 作为后续开发、联调、验收、复盘的依据

### 3. 使用原则
- **总纲是宪法**
- **附录是专项执行手册**
- 不允许只看附录不看总纲
- 不允许附录内容违背总纲原则
