# Phase 3 任务分工说明

**项目**: EDT指挥中心 - 网页可视化投研系统  
**核心理念**: 一个网页看全链路，新闻变→板块变→股票变→决策变  

---

## 一、用户故事

| 用户操作 | 系统响应 |
|----------|----------|
| 打开网页 | 显示实时新闻流（左栏） |
| 点击新闻 | 中栏显示受影响板块，右栏显示机会池 |
| 点击板块 | 右栏筛选该板块的优质股票 |
| 查看机会 | 显示入场区间、风险旗标、最终动作 |

---

## 二、系统架构

```

│  └─────────────┘   └─────────────┘   └─────────────────────┘  │
│       ↑                   ↑                   ↑                │
│       └───────────────────┴───────────────────┘                │

```

---

## 三、分工原则

- 三人三模块，**逻辑上互不依赖**，**通过事件契约交互**
- 各自独立开发、独立测试，最后通过WebSocket集成
- **任何字段变更必须四联动**: schemas + tests + module-registry.yaml + 文档
- **接口变更需通知下游模块Owner**

---

## 四、术语表

| 术语 | 定义 |
|------|------|
| **trace_id** | 事件追踪ID，贯穿全链路，由C模块(event_bus)统一生成 |
| **G7降级** | AI超时/错误时的安全降级策略，强制转为WATCH/BLOCK状态 |
| **差异化率** | 利好场景输出LONG的比例 + 利空场景输出SHORT的比例，需>=80% |
| **PENDING_CONFIRM** | 待人工确认状态，不可直接执行 |
| **BLOCK** | 被风控拦截，禁止执行 |

---

## 五、模块边界明确

### 边界归属定义
| 争议点 | 归属 | 说明 |
|--------|------|------|
| **trace_id生成** | C模块 | event_bus.py统一生成，保证全局唯一 |
| **风险旗标类型定义** | C模块 | 定义risk_flags类型清单（liquidity/volatility/market_cap等） |
| **新闻去重策略** | C模块 | 可配置去重参数，A模块只提供原始新闻 |
| **数据持久化方案** | C模块 | 统一管理消息存储（Redis/SQLite） |
| **配置热更新** | C模块 | 所有configs/*.yaml支持运行时加载 |

## 七、模块A：后端数据流（新闻→板块）

### A-1 任务：板块影响映射配置
- **交付物**: `configs/sector_impact_mapping.yaml`
- **验收**: 能输出板块方向(LONG/SHORT)与影响强度
- **配置来源**: 人工维护 + 规则引擎生成

### A-2 任务：新闻去重与聚类增强
- **交付物**: `NewsIngestion` 模块增强
- **验收**: 30分钟内同事件不重复显示，去重准确率≥95%
- **去重参数**: 可在C模块配置中心调整

### A-3 任务：传导链层级配置
- **交付物**: `configs/conduction_chain.yaml`
- **验收**: 展示"宏观→板块→主题"链路
- **层级结构**: macro → sector → theme

### A模块输入
```json
{
  "type": "news_input",
  "headline": "美联储暗示降息预期放缓",
  "source": "路透",
  "severity": "E3"
}
```

### A模块输出（给事件总线）
```json
{
  "type": "sector_update",
  "trace_id": "evt_xxx",
  "sectors": [
    {"name": "半导体", "direction": "LONG", "impact_score": 0.85, "confidence": 0.90},
    {"name": "新能源", "direction": "SHORT", "impact_score": 0.40, "confidence": 0.70}
  ],
  "conduction_chain": [
    {"level": "macro", "name": "原油", "relation": "affects_above"},
    {"level": "sector", "name": "能源", "relation": "same"},
    {"level": "theme", "name": "化工", "relation": "affects_below"}
  ],
  "timestamp": "2026-04-03T10:32:00Z"
}
```

### A模块复用
- `ai_event_intel.py`: 新闻摄取
- `intel_modules.py`: 情报处理
- `narrative_state_recognizer.py`: 叙事状态识别
- `conduction_mapper.py`: 传导映射

### A模块异常处理
| 异常场景 | 降级策略 |
|----------|----------|
| 新闻源不可用 | 切换Fallback新闻源 |
| 板块映射失败 | 返回空sectors，标记warning |
| AI超时 | 触发G7降级，输出WATCH |

### A模块验收
- [ ] 新闻点击后，板块更新延迟 ≤1s
- [ ] 方向明确（LONG/SHORT，不允许只有分数）
- [ ] 传导链完整展示（宏观→板块→主题）
- [ ] 同事件30分钟内不重复显示，去重准确率≥95%
- [ ] trace_id全程贯穿
- [ ] 板块覆盖度≥90%（已知板块都能映射）

## 八、模块B：策略层（板块→股票→机会）

### B-1 任务：优质股票池配置
- **交付物**: `configs/premium_stock_pool.yaml` + 筛选逻辑
- **验收**: 机会池仅输出优质池内股票
- **筛选指标**: ROE>15%、市值>500亿、流动性评分>0.6

### B-2 任务：多空机会评分规则
- **交付物**: `opportunity_score` 模块
- **验收**: 利好→LONG，利空→SHORT，方向一致性≥85%
- **差异化统计**: 统计100条利好新闻输出LONG的比例

### B-3 任务：机会卡字段补全
- **交付物**: 机会卡（入场区间/风险旗标/理由摘要）
- **验收**: 字段完整，可直接用于投研讨论

### B模块输入
- A模块输出的 `sector_update`

### B模块输出（给事件总线）
```json
{
  "type": "opportunity_update",
  "trace_id": "evt_xxx",
  "opportunities": [
    {
      "symbol": "688981",
      "name": "中芯国际",
      "sector": "半导体",
      "signal": "LONG",
      "entry_zone": {"support": 45.2, "resistance": 47.5},
      "risk_flags": [{"type": "liquidity", "level": "medium", "description": "流动性偏低"}],
      "final_action": "EXECUTE",
      "reasoning": "半导体板块强利好，个股基本面评分0.82，入场区间合理",
      "confidence": 0.85,
      "timestamp": "2026-04-03T10:32:05Z"
    }
  ]
}
```

### 机会卡字段说明
| 字段 | 说明 | 示例 |
|------|------|------|
| symbol | 股票代码 | 688981 |
| name | 股票名称 | 中芯国际 |
| sector | 所属板块 | 半导体 |
| signal | 信号方向 | LONG/SHORT/WATCH |
| entry_zone.support | 支撑位 | 45.2 |
| entry_zone.resistance | 阻力位 | 47.5 |
| risk_flags | 风险旗标数组 | [{"type":"liquidity", "level":"medium"}] |
| final_action | 最终动作 | EXECUTE/WATCH/BLOCK/PENDING_CONFIRM |
| reasoning | 理由摘要 | 50字以内 |
| confidence | 置信度 | 0.85 |

### B模块复用
- `market_validator.py`: 市场验证
- `signal_scorer.py`: 信号评分
- `ai_signal_adapter.py`: AI信号适配
- `decision_card.py`: 决策卡生成
- `fatigue_calculator.py`: 疲劳度计算

### B模块异常处理
| 异常场景 | 降级策略 |
|----------|----------|
| sector_update为空 | 机会池返回空，标记无信号 |
| AI信号超时 | 触发G7降级，返回WATCH |
| 优质股票池为空 | 返回空池，提示配置检查 |

### B模块验收
- [ ] 板块切换后，机会池更新延迟 ≤1s
- [ ] 利好→LONG，利空→SHORT，差异化率≥80%
- [ ] 机会池只显示优质池内股票
- [ ] 机会卡字段完整（入场区间+风险旗标+理由）
- [ ] 高风险自动BLOCK或PENDING_CONFIRM，拦截率100%
- [ ] 方向一致性≥85%（同板块内股票信号方向一致）

### 风控规则
```python
def risk_check(opportunity):
    # 高风险自动BLOCK
    if len(opportunity.risk_flags) >= 3:
        return "BLOCK"
        
    # 中风险进入待确认
    if any(f.level == "high" for f in opportunity.risk_flags):
        return "PENDING_CONFIRM"
        
    # AI超时/错误触发G7降级
    if opportunity.ai_timeout_or_error:
        return "WATCH"
        
    return opportunity.final_action
```

## 十一、事件契约（三人对接标准）

### 三类消息
| 消息类型 | 字段 | 说明 |
|----------|------|------|
| event_update | trace_id, headline, source, severity, evidence_score, narrative_state | 新闻事件 |
| sector_update | trace_id, sectors[].name/direction/impact_score/confidence, conduction_chain | 板块更新 |
| opportunity_update | trace_id, opportunities[].symbol/name/signal/entry_zone/risk_flags/final_action/reasoning | 机会更新 |

### 关键规则
1. 同一 `trace_id` 的消息按时间有序
2. A/B模块只管发消息，不关心谁接收
3. C模块负责订阅、路由、推送
4. **任何字段变更必须四联动**: schemas + tests + module-registry.yaml + 文档
5. **接口变更需通知下游模块Owner**
