## 二、阶段四目标

阶段四目标：把“可联调系统”升级为“可验收的生产前系统”。

### 验收口径（阶段四结束时）
- 数据真实性：真实源断开时必须显式失败（禁止静默产出）
- 裁决一致性：同事件跨实例输出一致
- 输出可靠性：支撑/阻力、机会评分基于实时价格
- 可审计性：关键决策可回放、可追溯、可归因

## 十、实施细则 SOP（防漂移执行版）

本节用于把“任务清单”扩写为可执行步骤。执行时遵循：
- 先改配置与契约，再改实现，再改测试与门禁。
- 每项任务必须附验证命令与回滚点。
- 未通过验证不得进入下一项。

### 10.1 P0-1 实时行情接入（VIX/SPX/last_price）

目标：移除 `unavailable/None` 行情，改为真实行情源。

涉及文件：
- `scripts/data_adapter.py`
- `scripts/realtime_news_monitor.py`
- `configs/edt-modules-config.yaml`

执行步骤：
1. 在配置中新增行情源参数（provider、timeout、retries、rate_limit）。
2. 在 `DataAdapter.fetch_market_data()` 实现真实行情拉取与字段标准化。
3. 在调用侧保留空值防御，但将 `is_test_data` 作为阻断标记。
4. 失败策略：行情不可用时返回 `market_data_source=failed`，并触发 `WATCH` 降级。

验证命令：
- `python3 scripts/run_intel_with_adapter.py`
- `python3 -m pytest -q tests/test_data_adapter.py`

通过标准：
- 同一标的不同时间请求，价格可变化。
- 不再出现固定 0.0 或长期 None 作为默认有效值。

回滚点：
- 保留 `feature flag`（如 `market_data.enabled`），失败可切回只读模式。

### 10.2 P0-2 fallback 强失败（源断开显式失败）

目标：真实新闻源不可用时，生产链路必须显式失败，禁止静默产出事件。

涉及文件：
- `scripts/ai_event_intel.py`
- `scripts/realtime_news_monitor.py`
- `scripts/system_healthcheck.py`

执行步骤：
1. 将 NewsIngestion 的 fallback 行为按环境隔离：`dev/test` 可用，`prod` 禁用。
2. `prod` 下源失败时返回 `FAILED` + 错误码（如 `NEWS_SOURCE_UNAVAILABLE`）。
3. 在 monitor 层接入告警日志与健康状态上报。
4. 补充测试：断网/超时场景下不产生下游事件。

验证命令：
- `python3 -m pytest -q tests/test_intel_modules.py`
- `python3 scripts/system_healthcheck.py`

通过标准：
- 生产模式源断开时，日志中有明确失败原因。
- 不产生 `event_update/sector_update/opportunity_update`。

回滚点：
- `allow_fallback_in_prod=false` 改为配置开关，紧急情况下可临时打开但必须告警。

### 10.3 P0-3 实时价格替换静态 last_price

通过标准：

### 11.1 能力增强方向（建议优先级）

| 优先级 | 方向 | 核心价值 | 交付物 |
|---|---|---|---|
| P1 | 策略后验归因层 | 从“推荐”升级为“可验证改进” | 命中率/盈亏比/误报率周报 |
| P1 | 自适应阈值层 | 提升极端行情下稳定性 | 动态阈值策略与回测报告 |
| P1 | 事件冲突仲裁层（总统决） | 避免同一时段多空互打 | 冲突仲裁规则与优先级引擎 |
| P2 | 模拟执行与风险预算层 | 从单票建议走向组合可执行 | paper-trade + 组合风险预算 |
| P2 | 运维质量面板 | 提升定位与运维效率 | 实时健康/延迟/误匹配可视化 |

### 11.2 详细说明

#### A. 策略后验归因层
- 目标：每条机会卡都有 T+1/T+5 结果回写。
- 指标：命中率、盈亏比、最大回撤、误报率。
- 建议文件：`scripts/attribution_engine.py`、`logs/attribution_ledger.jsonl`。

#### B. 自适应阈值层
- 目标：`WATCH/EXECUTE` 阈值按市场波动自动调整。
- 触发因子：VIX、板块波动、事件密度。
- 建议文件：`scripts/adaptive_thresholds.py`、`configs/adaptive_rules.yaml`。

#### C. 事件冲突仲裁层（总统决）
- 目标：同一窗口内多个相反事件统一仲裁。
- 仲裁因子：来源等级、时效、语义置信度、市场验证。

## 十三、增量修复记录（2026-04-09：板块别名字典 + Energy 候选恢复）

### 13.1 问题现象与根因
- 现象：页面出现 `Energy` 板块热力图，但“候选股票/机会卡”为空。
- 根因：B 层 `OpportunityScorer` 的回退选股按板块名匹配，输入为英文 `Energy`，股票池配置为中文 `能源`，导致 `pick_by_sector()` 无命中。

### 13.2 解决方案（配置化别名字典）
- 新增配置：`configs/sector_aliases.yaml`，将中英板块名统一归一（如 `Energy -> 能源`）。
- 代码改造：`scripts/opportunity_score.py`
  - 新增 `SectorAliasResolver`，加载并解析别名字典。
  - `PremiumStockPool.pick_by_sector()` 使用 canonical sector 匹配。
  - `OpportunityScorer.build_opportunity_update()` 在 `stock_candidates` 分组和 `sectors` 查找时统一 canonical key。

### 13.3 测试与验证证据
- 新增回归测试：`tests/test_opportunity_score.py::test_fallback_pool_supports_sector_alias_dictionary`
  - 用例：输入板块 `Energy` 且无 seeded candidates，断言可回退命中 `XOM`。
- 验证结果：
  - `python3 -m pytest tests/test_opportunity_score.py -k sector_alias_dictionary -v` -> `1 passed`
  - `python3 -m pytest tests/test_conduction_mapper_dynamic.py tests/test_opportunity_score.py -v` -> `12 passed`

### 13.4 影响范围
- 不改变机会卡字段 schema；仅增强板块名归一能力。
- 对已有中文链路无破坏；对英文实时板块输入（如 `Energy`, `Technology`, `Financial Services`）提升命中率。
